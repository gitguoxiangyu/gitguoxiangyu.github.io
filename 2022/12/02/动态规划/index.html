初学动态规划

动态规划

动态规划，英文：Dynamic Programming，简称DP，如果某一问题有很多重叠子问题，使用动态规划是最有效的。

所以动态规划中每一个状态一定是由上一个状态推导出来的，这一点就区分于贪心，贪心没有状态推导，而是从局部直接选最优的

简单来说，贪心是不需要考虑外部状态，而动态规划需要考虑上下文的数据状态

动态规划的一般步骤

  • 第一步：确定dp数组的行列数和下标的具体含义
  • 第二步：确定dp的推导公式
  • 第三步：确定dp数组的初始化数据
  • 第四步：确定遍历顺序
  • 第五步：推导dp数组

爬楼梯（基础题）

  • 题目：

    假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

    每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

    注意：给定 n 是一个正整数。

  • 思路：

    n >= 2 时，想要到达这阶楼梯，只有两种情况，在 n-1 阶楼梯，爬一步楼梯；在 n-2 阶楼梯，爬两阶楼梯

    所以，到达 n 阶楼梯的方法数 = 到达 n-1 阶楼梯的方法数 + 到达 n-2 阶楼梯的方法数

    所以，可以推导dp的递推公式为：dp[n] = dp[n-1] + dp[n-2]

    显然可知，dp[1] = 1，dp[2] = 2

  • 代码：

    时间复杂度O(n)，空间复杂度O(n)

不同路径（基础题）

  • 题目：

    一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。

    机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。

    问总共有多少条不同的路径？

  • 思路：

    初始化 dp数组 为一个二维 m x n 数组，dp[i][j] 的含义为到达该格子的方法数

    想要到达某个特定的格子，只能先到达其上方或者左方的格子

    因此到达每一个非左边缘且非上边缘格子的方法数 = 到达该格子左边的格子方法数 + 到达该格子上方的格子方法数（ 类似爬楼梯 ）

    一个3x7的示例

    所以，可以推导dp的递推公式为：dp[i][j] = dp[i-1][j] + dp[i][j-1]

    因为机器人每次只能向下或者向右移动一步，因此到达左边边缘和上边边缘的格子的方法数为1

  • 代码：

    时间复杂度O(m x n)，空间复杂度O(m x n)

背包问题（进阶题）

  • 题目：

    有n件物品和一个最多能背重量为w 的背包。第i件物品的重量是weight[i]，得到的价值是value[i] 。每件物品只能用一次，求解将哪些物品装入背包里物品价值总和最大。

  • 思路：

    对每个不同的物品而言，我们需要对其进行决策——放入背包或不放入背包，且每次决策都依据上一次决策的结果

    初始化 dp数组 为一个二维 n x w 数组，dp[i][j] 的含义为该决策情况下的物品价值，第 i 行代表第 i 个物品，第 j 列代表当前背包重量为 j

    想要确定 dp[i][j] 的物品价值，就需要确定物品放入的情况

    对每个物品 i 而言，只有放入和不放入两种情况：

    当放入时，该格的物品价值 = 该格上一行向左 weight[i] 格的物品价值（即上一件物品，且减去当前物品的重量） + 当前物品的价值

    dp[i][j] = dp[i - 1][j - weight[i]] + value[i]

    当不放入时，该格的物品价值 = 该格上一格的物品价值（即上一件物品，且背包容量不变）(其实就是当物品i的重量大于背包j的重量时，物品i无法放进背包中，所以被背包内的价值依然和前面相同。)

    dp[i][j] = dp[i - 1][j]

    我们需要背包价值最大，所以取二者中更大的

    所以，可以推导dp的递推公式为：dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - weight[i]] + value[i])
